რა არის Protocol:

პროტოკოლი არის წესების ნაკრები რომლებიც განსაზღვრავენ თუ როგორ უნდა გაიცვალოს ინფორმაცია კომპიუტერებს შორის

რა არის HTTP:
მარტივად რომ ვთქვათ ეს არის პროტოკოლი პროტოკოლი არის წესების ნაკრები მაგრამ წარმოვიდგინოთ რომ არის ენა რომელზეც კომპიუტერები საუბრობენ ერთმანეთში, კომპიუტერს და ვებ სერვერს ჭირდებათ სტანდარტული წესების ნაკრები რათა გაცვალონ ინფორმაცია

HTTP რექუესტი შეიცავს დეტალებს თუ რაგინდა და რას ითხოვ შენ ვებ სერვერისგან ეს შეიძლება იყოს მაგალთად იუთბის ჰოუმ ფეიჯი ან რაიმე სხვა ვიდეოს ლინკი


HTTP განსაზღვრავს წესებსა და ფორმატებს, თუ როგორ ხდება შეტყობინებების აგება, გაგზავნა და ინტერპრეტაცია კლიენტს (როგორც თქვენი ბრაუზერი) და სერვერს შორის. ის სტანდარტიზებს კომპიუტერების ერთმანეთთან კომუნიკაციას ინტერნეტის საშუალებით, რაც უზრუნველყოფს ინფორმაციის გაცვლის საერთო ენას.


საჯარო გასაღები: ეს გასაღები ღიად არის გაზიარებული და გამოიყენება დაშიფვრისთვის საპირისპირო მხარის მიერ. მაგალითად, სერვერის საჯარო გასაღები შედის მის ციფრულ სერთიფიკატში და უზიარებს კლიენტს.

პირადი გასაღები: ეს გასაღები საიდუმლოდ ინახება და ცნობილია მხოლოდ მისი მფლობელი მხარისთვის. HTTPS-ის კონტექსტში, სერვერი უსაფრთხოდ ინახავს თავის პირად გასაღებს და იყენებს მას შესაბამისი საჯარო გასაღებით დაშიფრული ინფორმაციის გასაშიფრად.

HTTPS key exchange:

Client-Server Handshake:
step1: 
ბრაუზერი აგზავნის რექუსეტს
step2:
სერვერი პასუხობს მას და უგზავნის სერთიფიკატს. ეს სერთიფიკატი შეიცავს სერვერის ფაბლიქ ქის და ასევე სხვა დამატებეთ ინფორმაციას რათა ამოიცნოთ რომ ეს სერვერი სანდოა
step3:
კლიენტი ვერიფიკაციას უკეთებს ამ სერთიფიკატს და წრმუნდება რომ მართალია

Key Exchange:
step4: 
კლიენტი ქმნის უნიკალური სიმეტრიულ ქის (გამოიყენება მონაცემების დასაშიფრად სესიის დროს)
step5:
სერვერის საჯარო გასაღების გამოყენებით ამოღებული სერთიფიკატიდან, კლიენტი შიფრავს ამ სიმეტრიულ გასაღებს და დაცულად უგზავნის სერვერს
step6:
სერვერს რომელსაც გააჩნია შესაბამისი პირადი გასაღები, შიფრავს ამ დაშიფრულ სიმეტრიულ გასაღებს

Secure Data Transfer:
step7:
ორივე სერვერიც და კლიენტიც იყენებს ამ სიმეტრიულ გასაღებს მონაცემების გაშიფრისთვის და დაშიფრისთვის, მონაცემების გაცვლის დროს
step8:
ინფორმაცია რომელიც იცვლება კლიენტს და სერვერს შორის არის დაშიფრული სიმეტრიული გასაღების მიერ რათა უზრუნველყოს კონფიდიცალურობა და მთლიანობა.

მხოლოდ სერვერს აქვს ფაბლიქ და ფრაივეით ქეიები

the server has both a public and a private key, but only the public key is shared with clients through its digital certificate. The client uses the server's public key to encrypt data (like the symmetric key), which the server can then decrypt using its private key. This asymmetric encryption and decryption process facilitates secure key exchange between the client and the server in HTTPS.





TCP(Transmission Control Protocol): გადაცემის კონტროლის პროტოკოლი, მოკლედ ეს ადგენს წესებს თუ როგორ უნდა მოხდეს ერთი კომპიუტერიდან მეორე კომპიუტერზე ინფორმაციის გადატანა, ეს შენს ინფორმაციას რომელსაც მიაწვდი დახლეჩს ძაან პატარ-პატარა პაკეტებად და შემდეგ ხდება ამ პაკეტების გაგზავნა და ეს ხდება გამგზავნის მხრიდან ხოლო მიმღების მხრიდან ეს აწყობს ამ პაკეტებს და გვაქ უკვე ის ინფორმაცია რაც გააგზავნეს

IP(Internet Protocol): მარტივად რო ავხსნათ ეს არის უკვე ის პროტოკოლი რომლითც ჩვენ ვაგნებთ ამა თუ იმ კომპიუტერს ქსელში დიდ ქსელში

git remote -v გვაჩვენებს დაკავშირებული ვართ თუ არა რომელიმე რეპოზიტორიასთან

git remote add <name> <URL> ვამატებთ ახალ რეპოზიტორიას თავისი სახელით და ლინკით

git remote remove <name> ვშლით დაკავშირებულ რეპოზიტორიას ლოკალურად სახელის მიხედვით, რეპოზიტორია იშლება მოხლოდ ლოკალური ფაილისთვის და არა გითჰაბზე

git remote rename <old_name> <new_name> სახელის გადარქმევა

git remote show <name> გვაჩვენებს რაღაც დამატებით ინფორმაციას ამ რემოუთზე

git push <remote_name> <branch> ვტვირთავთ ლოკალურ კომიტებს რემოუთ რეპოზიტორიაში, git push origin main

git pull <remote_name> <branch> ამით მოგვაქ რემოუთ საქაღალდე git pull origin main


როცა უკვე ავტვირთავთ ჩვენს ფაილებს გითჰაბზე და შემდეგ შევცვლით მათ ყოველთვის გვჭირდება გავაკეთოთ ახლიდან ფუში რათა განახლდეს იქაც ჩვენი ცვლილებები

git clone sshkey an link ამით შეგვიძლია კლონირება გავაკეთოთ რემოუთ საქაღალდის ამაში გაერთიანებულია სამი ქომანდი
git init, git remote add, git pull

SSH key: ეს არის დაცული გზა რომ ვიკომუნიკაციოთ რემოუთ სერვერთან, დაშიფრული კომუნიკაციით

საჯარო გასაღები : ეს გასაღები გაზიარებულია სხვა სისტემებთან ან სერვერებთან, რომლებზეც გსურთ წვდომა. ის მოთავსებულია დისტანციურ სერვერზე, რომელთანაც გსურთ დაკავშირება. საჯარო გასაღები გამოიყენება მონაცემების დაშიფვრად, რომელთა გაშიფვრა შესაძლებელია მხოლოდ შესაბამისი პირადი გასაღებით.

პირადი გასაღები : ეს გასაღები უსაფრთხოდ ინახება თქვენს ადგილობრივ სისტემაში. ის გამოიყენება მონაცემთა გასაშიფრად, რომელიც დაშიფრულია ასოცირებული საჯარო გასაღებით. ის არასდროს არ უნდა იყოს გაზიარებული ან გამჟღავნებული.
